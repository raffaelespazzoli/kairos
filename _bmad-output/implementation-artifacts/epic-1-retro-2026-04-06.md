## Successes

- **100% story completion** in 4 days -- exceptional velocity for a foundation epic
- **Foundation patterns** established in Story 1.5 (`apiFetch`, `useApiFetch`, `ErrorAlert`, `LoadingSpinner`, `RefreshButton`) became the standard toolkit that accelerated all subsequent stories
- **Test coverage grew steadily** from 0 to 210+ tests with zero regressions across the entire epic
- **Story 1.6 (Vault Credential Provider)** was the cleanest implementation -- zero debug issues, zero review findings -- evidence of the team learning curve paying off
- **Code reviews** caught 9 real gaps across Stories 1.3 and 1.5 (e.g., "works in isolation but not wired end-to-end"), all addressed
- **Adapter pattern + PortalIntegrationException** convention proved its value -- once mature, new adapters were nearly frictionless

## Challenges

- **Repeated gotchas:** PanacheEntity SEQUENCE vs IDENTITY strategy hit in both Story 1.2 and Story 1.7; PatternFly 6 API differences from PF5 surfaced in Stories 1.4, 1.5, and 1.7. Knowledge existed in story dev notes but didn't propagate to `project-context.md` quickly enough
- **Test infrastructure friction:** Story 1.3 discovered endpoints must exist for JAX-RS filters to fire (required stub resources); Story 1.5 found PermissionFilter blocking test stubs under `/api/v1/` (required relocation to `/test/`)
- **Dependency version defaults:** `create-vite@latest` installs React 19, Vite 8, and router v7 by default -- all wrong for the project. Required explicit version pinning

## Key Insights

1. **Foundation patterns pay compounding dividends** -- the time invested in Story 1.5's error/loading/refresh toolkit saved effort on every subsequent story
2. **Knowledge must live in `project-context.md`** to prevent repeat gotchas -- story-level dev notes aren't sufficient for cross-story learning by AI agents
3. **Code reviews catching "not wired end-to-end" gaps** are high-value and prevent technical debt from accumulating into later epics
4. **Test coverage built incrementally** (0 → 210+) with zero regressions demonstrates disciplined engineering and catches issues early
